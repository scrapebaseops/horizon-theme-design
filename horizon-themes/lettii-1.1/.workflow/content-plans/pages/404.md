# 404 Error Page Spec

## Page Metadata
- **Template**: `templates/404.json`
- **Layout**: Default (standard header + footer)
- **URL**: Any non-matching URL (e.g., `/anything-that-doesnt-exist`)
- **Page Title**: "Page not found | Lettii"
- **Purpose**: Recovery page that guides lost visitors back to useful content while maintaining brand warmth
- **Priority**: Standard (required Shopify template)

## Target Audience
- Visitors who followed a broken link from ads, social, email, or external sites
- Returning visitors who bookmarked a product or page that has since moved
- Search engine crawlers indexing removed URLs
- Most likely arriving from Meta/Google ads with a mistyped or expired link

## Why This Page Matters
A 404 is a friction point that can lose a sale. For a new brand like Lettii where every ad click costs money, recovering lost visitors is a direct revenue concern. The page must feel intentional, not broken.

## Sections

### Section 1: Recovery Message
**Shopify Section Type**: `main-404` (Horizon built-in)
**Purpose**: Acknowledge the error, maintain brand voice, provide immediate recovery paths

**Draft Content**:

**Heading (h1)**:
> We can't find that page. But we can light the way back.

**Body text (paragraph)**:
> The page you're looking for may have moved or no longer exists. Try searching for what you need, or explore our collections below.

**Primary CTA button**:
> Continue shopping

Button links to: `/collections/all-lamps` (All Lamps collection)

**Content Sources**:
- Heading: Static text, configured in theme editor via text block
- Body: Static text, configured in theme editor via text block
- Button: Static, configured in theme editor via button block with link to collection

**Configurable Settings**:
| Setting | Type | Default | Notes |
|---------|------|---------|-------|
| Heading text | Rich text (text block) | "We can't find that page. But we can light the way back." | Merchant-editable |
| Body text | Rich text (text block) | "The page you're looking for may have moved..." | Merchant-editable |
| Button label | Text | "Continue shopping" | Merchant-editable |
| Button link | URL | `/collections/all-lamps` | Merchant-editable |
| Section height | Select | `small` | Auto / Small / Medium / Large / Full screen |
| Alignment | Select | `center` | Left / Center / Right |
| Color scheme | Color scheme picker | Default (scheme-1) | Follows theme color schemes |
| Padding top | Range (0-100px) | `100` | Desktop spacing |
| Padding bottom | Range (0-100px) | `100` | Desktop spacing |

**Responsive Notes**:
- **Desktop**: Centered text stack, generous vertical padding (100px top/bottom), max-width constrains text for readability
- **Mobile**: Same centered layout, text remains legible at mobile widths, padding scales proportionally

**Media Requirements**: None. This is a text-only section. Clean and fast-loading.

**Shopify Section Type**: `product-list` (Horizon built-in, labeled "Featured Collection" in editor)
**Purpose**: Show products from a collection to re-engage visitors and recover potential purchases

## Shared Components

### Header
Standard site header with:
- Logo, navigation, search icon, cart icon
- Announcement bar with rotating trust messages (30-day guarantee, free shipping over 50, free returns, USB-C rechargeable)
- Sticky on scroll

The header provides the **search functionality** referenced in the requirements. Visitors can use the search icon in the header to find what they were looking for. No additional inline search bar is needed in the 404 section itself -- the header search is always accessible and keeps the page clean.

### Footer
Standard site footer with:
- 4-column navigation (Shop, Help, About, Business)
- Newsletter signup with 10% off incentive
- Social links (Instagram, Pinterest, Facebook)
- Payment icons, policy links, copyright
- Trust signals embedded in footer layout

## Content Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Tone | Warm, helpful, not apologetic | Lettii's voice is confident. "We can't find that page" is honest without grovelling. "Light the way back" is on-brand (lighting company). |
| Search bar placement | Via header search icon (not inline) | Horizon theme has search built into the header. Adding a redundant inline search would require custom development for minimal gain. Header search is visible and accessible. |
| Collection links vs. product grid | Product grid from featured collection | Products are more engaging and actionable than text links. Showing actual lamps creates purchase intent. The product-list section is a native Horizon component. |
| Number of products | 4 | Enough to browse, not enough to overwhelm. Fits a clean 4-column desktop grid and 2-column mobile grid. |
| Featured collection | All Lamps (default) | Broadest selection. Merchant can change to Best Sellers or a seasonal collection post-launch. |
| No lifestyle imagery | Intentional | Keeps the page lightweight and fast. The product grid provides visual interest. Hero images on a 404 would feel overdone for a utility page. |
| "Continue shopping" as primary CTA | Standard Shopify convention | Visitors understand this immediately. Links to All Lamps rather than homepage -- gets them closer to products. |

## SEO

- **Page title**: "Page not found | Lettii" (set via theme or Shopify defaults)
- **HTTP status**: 404 (handled by Shopify automatically)
- **Meta robots**: `noindex` (Shopify handles this for 404 pages by default)
- **No canonical tag** needed (404 pages should not be indexed)
- **Internal linking value**: The product grid and header/footer navigation provide crawl paths back to important pages

---

## Accessibility

- **Heading hierarchy**: Single `h1` for the error message, `h4` for the product grid heading
- **Button semantics**: "Continue shopping" uses a proper `<a>` element styled as a button, with meaningful link text
- **Focus management**: No special focus management needed (standard page load)
- **Color contrast**: Inherits from theme color scheme; ensure error text meets WCAG 2.1 AA (4.5:1 contrast ratio)
- **Screen reader**: The h1 clearly communicates "page not found" context; product cards include product titles and prices as text
- **Keyboard navigation**: All interactive elements (button, product cards, header search, footer links) are keyboard-accessible via Horizon defaults

---

## Implementation Complexity

**Complexity**: Low

This page uses two native Horizon sections (`main-404` and `product-list`) with no custom development required. Implementation is entirely through the Shopify theme editor:

1. Edit the `main-404` section text blocks to use Lettii's copy
2. Add a `product-list` section below with the "All Lamps" collection selected
3. Configure grid settings (4 columns, 4 products)

**Estimated setup time**: 10-15 minutes in theme editor

**Template JSON structure** (for reference):
```
sections:
  main (type: main-404)
    blocks: text (h1), text (paragraph), button
  product_list (type: product-list)
    blocks: header text, product card config
order: [main, product_list]
```
